Learning Rate ? Learning Rate
Controls how fast the model updates its weights each step. Uses a cosine schedule: starts at peak (2e-4), gradually decays to near zero.
Smooth cosine decay is expected
Warmup phase rises in the first few steps
Too high = unstable loss spikes
Too low = model stops learning

Epoch Progress ? Epoch Progress
One epoch = one full pass through the entire training dataset. Training for 2 epochs means the model sees every example twice. More epochs improve learning but too many risk overfitting (memorizing rather than generalizing).
1–3 epochs is typical for LoRA fine-tuning
>5 epochs risks overfitting on small datasets

Training Loss ? Training Loss Over Time
Should show a smooth downward curve. Small spikes are normal, especially early on. If loss plateaus, the learning rate may be too low or the model has converged. If loss increases after decreasing, the model may be overfitting.
Learning Rate Schedule ? LR Schedule (Cosine Decay)
This chart should show a smooth cosine curve — rising during warmup steps, peaking, then gradually decaying toward zero. This is expected behavior and is not a problem. The cosine schedule prevents abrupt learning rate changes that could destabilize training.
Gradient Norm ? Gradient Norm
Measures the magnitude of weight updates per step. Indicates training stability.
0.1–5.0 — Stable, healthy gradients
5–15 — Acceptable fluctuations
15–50 — Worth monitoring
>50 — Exploding gradients, reduce LR
GPU Memory (GB) ? GPU Memory Over Time
Should remain stable with minor fluctuations during training. A gradual upward trend could indicate a memory leak. Sudden drops happen during checkpointing when optimizer states are temporarily offloaded.
