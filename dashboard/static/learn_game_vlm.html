Phase 4 — Vision

Game VLM Adapter

Train a vision-language model to play Pac-Man. The observe-reason-act loop you build here is the same architecture that drives TARS robotics and robot navigation.

1 Why Game VLM?

The visual decision-making loop — observe frame, reason, act — is the same architecture needed for TARS robotics and robot navigation. A robot sees camera frames, reasons about obstacles and goals, then issues motor commands. That's exactly what a game-playing VLM does.

Pac-Man is the training ground: simple enough to learn fast, complex enough to require real spatial reasoning. The maze has structure, ghosts create dynamic threats, pellets demand path planning. If the model can navigate a Pac-Man board, the same visual grounding transfers to navigating a warehouse floor.

This phase bridges the gap between text-only fine-tuning (Phases 0-3) and embodied AI. You already know how to train language models. Now you teach one to see.

2 Vision-Language Models Explained
How VLMs Work

A VLM combines two systems: an image encoder (vision tower) that converts pixels into "visual tokens", and a language model that processes those tokens alongside text. The key insight is a projection layer that maps visual features into the same embedding space as text tokens. Once projected, the LLM can't tell the difference — images become just another sequence of tokens.

Why This Matters

Traditional computer vision outputs labels or bounding boxes. A VLM outputs language — it can explain what it sees, reason about spatial relationships, and justify its decisions. That reasoning chain is what makes game-playing (and eventually robot navigation) possible.

3 Qwen3.5-9B Architecture
Model Overview

9 billion parameters with built-in vision capabilities. Unlike some VLMs that bolt a vision encoder onto a text-only model, Qwen was designed from the ground up to handle multimodal input. The vision tower processes images at configurable resolution with min_pixels: 50,176 and max_pixels: 100,352.

Key Features

Multi-image input — the model can process multiple frames in a single prompt, enabling temporal reasoning across game states. This is critical for understanding movement direction and predicting ghost trajectories.

Configurable resolution — reducing max_pixels speeds up training significantly. Pac-Man frames don't need high-res detail; the maze structure and entity positions are clear even at lower resolution.

Why Qwen Over Other VLMs

Qwen was chosen for several reasons: native vision support (no adapter hacks), efficient token usage (visual tokens are compressed well), strong spatial reasoning in benchmarks, and 9B is trainable on DGX Spark with LoRA. Larger VLMs like LLaVA-34B would require multi-GPU setups. Smaller ones like LLaVA-7B lack the reasoning depth for complex game states.

System Prompt
You are TARS, playing Pac-Man. Analyze the game frame and choose an action. ACTIONS: NONE (continue current direction), UP, DOWN, LEFT, RIGHT Respond with brief reasoning followed by: Action: <ACTION>

The model must output reasoning first, then a structured action. This chain-of-thought format forces the model to articulate what it sees before deciding, which improves action quality and makes failures debuggable.

Dataset Structure

Frame images paired with expert actions. Training examples are organized as episode-based sequences for temporal learning — the model sees frames in the order they occurred during gameplay, building an implicit sense of movement and game flow.

5 LoRA for VLMs
Same Concept, Different Application

LoRA (Low-Rank Adaptation) works the same way as in text-only fine-tuning: inject small trainable matrices into the model's attention layers while keeping the original weights frozen. The key difference for VLMs: LoRA is applied to the language model layers only, not the vision encoder. The vision tower stays frozen — it already knows how to see. We're teaching the LLM how to reason about what it sees.

Configuration
LoRA Config: r: 32 # Rank (higher than text SFT) alpha: 64 # Scaling factor (2x rank) targets: - q_proj # Query projection - k_proj # Key projection - v_proj # Value projection - o_proj # Output projection - gate_proj # MLP gate - up_proj # MLP up - down_proj # MLP down
Why Higher Rank?

Text SFT typically uses r=16. Game VLM uses r=32. Visual reasoning is more complex than text pattern matching — the model needs more trainable parameters to learn the mapping from spatial features to appropriate actions. The higher rank gives the adapter more capacity to capture the visual-to-action relationship.

6 Dataset Structure
File Layout

Frame images are stored in data/games/pacman/. Each training example is a conversation pair: the user provides an image with "analyze this frame", and the assistant responds with reasoning followed by an action.

Training example format: User: [image] Analyze this game frame and choose an action. Assistant: Pac-Man is heading right in the middle corridor. A ghost is approaching from above. The nearest pellet cluster is to the left. Action: LEFT
Action Balance

Without balancing, the dataset would be dominated by NONE actions (continue current direction). In most frames, the optimal move is to keep going. But if the model learns to always say NONE, it never turns — useless.

Two mechanisms fix this:

max_noop_ratio=0.25 — caps NONE actions at 25% of the dataset. Excess NONE frames are discarded.

decision_point_oversample=3 — frames where the expert changes direction are duplicated 3x. These are the critical moments: turns, ghost avoidance, pellet pursuit. The model sees them more often.

Without balancing, NONE would dominate at ~60%. Capping and oversampling create a healthy distribution.
Training & Evaluation
7 Training Walkthrough
Running Training
cd games bash train_game.sh # Or directly: python train_game_adapter.py
What to Watch

Loss should decrease, but VLM training is noisier than text-only. Each batch contains images of different game states with different optimal actions — the gradient signal is inherently more variable.

Image processing is slower per step. Each training step must encode frames through the vision tower before the LLM forward pass. Expect 3-5x slower steps compared to text-only SFT at the same sequence length.

Reduced pixel bounds speed things up. The configurable min_pixels/max_pixels in the Qwen config control how many visual tokens each image produces. Fewer pixels = fewer tokens = faster training, at the cost of visual detail. For Pac-Man's simple graphics, the reduced bounds are sufficient.

Expected Training Curve

Unlike text SFT where loss drops smoothly, VLM training often shows plateaus followed by sudden drops. The model may spend many steps learning basic visual parsing before suddenly "getting" spatial relationships. Don't panic if loss stalls for a while — that's normal for visual tasks.

8 Evaluation
Running Eval
python games/eval_game_adapter.py
What to Look For

Action appropriateness — does the model choose sensible actions given the game state? Is it turning away from ghosts? Moving toward pellets?

Reasoning quality — does the model explain what it sees? A good response identifies Pac-Man's position, nearby threats, and pellet locations before choosing an action. A bad response gives a vague justification or ignores visible threats.

Common Failure Modes

Directional bias — the model favors one direction (e.g., always RIGHT). This usually means the dataset wasn't balanced well, or the model hasn't trained long enough to learn spatial nuance.

Ignoring ghosts — the model walks straight into threats. The vision encoder may be struggling to distinguish ghost sprites from background elements. Check that frame resolution is sufficient.

Not pursuing pellets — the model wanders aimlessly instead of path-planning toward pellet clusters. This is a higher-order reasoning failure that typically improves with more training steps.

Visual Grounding

Visual grounding — connecting what you see to actions — is the fundamental capability. In Pac-Man, "ghost approaching from the left" maps to "go right." In a warehouse, "obstacle on the left" maps to "steer right." The spatial reasoning is identical; only the vocabulary changes.

What Transfers

Pac-Man maze → Robot navigation: walls become physical obstacles, corridors become hallways.

Ghost threats → Dynamic obstacles: moving objects that require avoidance planning.

Pellet collection → Task objectives: navigate to goal locations efficiently.

Decision points (turns) → Intersection handling: where the model must choose between paths.
